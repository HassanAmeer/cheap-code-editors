# UI Streaming Skill Restrictions

1. **Performance**: Always use `process.stdout.write` directly for text streaming. DO NOT use `log-update` for streamed text as it drastically slows down terminal rendering and introduces frame drops.
2. **Duplication**: Never print the AI response twice. Stream the response once directly to the console.
3. **Sticky UI Elements**: The input field prompt must remain fixed at the bottom of the screen. Above it, there MUST be an animated word reflecting the current action (e.g., "Thinking...", "Planning...", "Executing..."). **DO NOT** hardcode "Streaming Response". Use the state's current spinner text or dynamic wording. Use terminal escape sequences (`\x1b[A\x1b[G\x1b[J`) to redraw these lines seamlessly. Ensure `cols - 1` is used for full-width elements to prevent terminal line-wrapping bugs that break erasure. **CRITICAL**: The footer MUST NEVER blink, hide, or freeze. Use a dedicated render loop (`setInterval` at ~20FPS) to erase and redraw the footer independently of stream network chunks. Network chunks should append to a print queue that is flushed by this render loop. This ensures the UI remains smooth and visible even during API delays.
4. **Thinking Tags**: Extract `<thinking>` and `<thought>` tags during the stream. Print thinking content in a dim/grey color without a border **ONLY if `process.env.IS_DEBUG === 'true'`**. The final response should be printed inside standard left-bordered boxes (`▌ `).
